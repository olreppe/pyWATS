# Changelog

All notable changes to pywats-client-service will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [2.0.0] - TBD

### Added
- Core client service architecture
- Converter framework with automatic discovery
- Queue service with automatic retry logic
- Connection service with health monitoring
- File watcher service for automatic data processing
- Configuration management system
- Multiple instance support
- Async/await throughout for better performance
- Comprehensive error handling and logging

### Changed
- Refactored from monolithic client to service-based architecture
- Improved converter plugin system
- Enhanced queue processing with better state management
- Modernized configuration format (JSON-based)

### Fixed
- Race conditions in queue processing
- Connection stability issues
- Converter loading edge cases

## [1.0.0] - Initial Release

### Added
- Initial release of client services
- Basic converter system
- Queue management
- WATS connection handling
