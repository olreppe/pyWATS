## üéØ Framework Design Principles

### Technology Alignment (Current GUI)
- **Use qasync** - Bridge Qt event loop with asyncio (already proven in production)
- **AsyncHelper** - Simplified AsyncAPIRunner for async API calls from Qt slots
- **Error Handling** - Reuse ErrorHandlingMixin patterns (standard error dialogs)
- **IPC Client** - Use AsyncIPCClient for service communication
- **Themes** - Support QSS stylesheets (can reuse from current GUI)
- **Single-Instance** - Use QLocalSocket/QLocalServer pattern (optional)

### Base Classes (Lightweight)
- `BaseApplication`: QApplication + qasync event loop setup
- `BaseMainWindow`: QMainWindow with standard menus/status bar
- `AsyncHelper`: Run async coroutines from Qt slots (simplified AsyncAPIRunner)
- `ErrorDialog`: Standard error display (reuse ErrorHandlingMixin patterns)

### Technology Reuse Pattern
**Reuse existing components from `pywats_client/gui/`:**
- **AsyncAPIRunner** ‚Üí Simplify for new framework (AsyncHelper)
- **ErrorHandlingMixin** ‚Üí Reuse or adapt for error dialogs

- **Styles (DARK_STYLESHEET)** ‚Üí Reuse QSS themes
- **IPC Client** ‚Üí Use AsyncIPCClient as-is for service communication
