Creating a UUT Report

A UUT (Unit Under Test) report contains all the test data for a single tested unit. Before creating a UUT report, you need to gather the required header information from your test data.

Required Information

To create a UUT report, you need the following information:

  • Operator Name - The person or system that performed the test
  • Part Number - The product/part identifier (SKU)
  • Part Revision - The revision of the part (e.g., "A", "1.0", "Rev B")
  • Serial Number - The unique identifier for this specific unit
  • Operation Type - The test process (retrieved from WATS server)
  • Sequence Name - The name of the test sequence
  • Sequence Version - The version of the test sequence

Getting the Operation Type

The Operation Type must be retrieved from the WATS server. Each test process in your WATS system has a unique operation type code (e.g., "10" for SW-Debug, "30" for ICT, "40" for Final Test). 

// Get operation type by code
OperationType? operationType = api.GetOperationType("30");

if (operationType == null)
{
    throw new Exception("Operation type '30' not found on WATS server!");
}

Console.WriteLine($"Using: {operationType.Name} (Code: {operationType.Code})");

Important: Always verify that GetOperationType() returns a valid result. If it returns null, the operation type code is not configured in your WATS Control Panel.

Creating the UUT Report Object

Use the CreateUUTReport() method to create a new UUT report:

using Virinco.WATS.Interface;

// Get operation type from server
OperationType? operationType = api.GetOperationType("30");

// Create the UUT report
UUTReport uut = api.CreateUUTReport(
    operatorName: "John Smith",
    partNumber: "PCB-1234",
    partRevision: "B",
    serialNumber: "SN123456789",
    operationType: operationType,
    sequenceFileName: "Final Test",
    sequenceFileVersion: "2.5"
);

Setting Additional Header Properties

After creating the UUT report, set additional properties as needed:

// Set test start time
uut.StartDateTime = DateTime.Parse("2026-02-07 14:30:00");

// Set test execution time (in seconds)
uut.ExecutionTime = 125.5;

// Set the test station name
uut.StationName = Environment.MachineName; // Or custom name

// Set batch/lot number (optional)
uut.BatchSerialNumber = "LOT2024-001";

// Set work order number (optional)
uut.Purpose = "WO-12345";

// Set UUT status (will be set automatically based on test results, but can override)
uut.Status = UUTStatusType.Passed; // or UUTStatusType.Failed

UUT Status Types

The UUT status indicates the overall test result:

  • UUTStatusType.Passed - All tests passed
  • UUTStatusType.Failed - One or more tests failed
  • UUTStatusType.Error - Test encountered an error
  • UUTStatusType.Terminated - Test was terminated before completion
  • UUTStatusType.Running - Test is in progress (rarely used)

Complete Example

using System;
using Virinco.WATS.Interface;

// Initialize API
TDM api = new TDM();
api.InitializeAPI(false);
api.ValidationMode = ValidationModeType.AutoTruncate;

// Get operation type
OperationType? operationType = api.GetOperationType("30");
if (operationType == null)
{
    throw new Exception("Operation type not found!");
}

// Create UUT report
UUTReport uut = api.CreateUUTReport(
    operatorName: "Auto",
    partNumber: "PCB-1234",
    partRevision: "B",
    serialNumber: "SN123456789",
    operationType: operationType,
    sequenceFileName: "Final Test",
    sequenceFileVersion: "2.5"
);

// Set additional properties
uut.StartDateTime = DateTime.Now;
uut.StationName = Environment.MachineName;
uut.ExecutionTime = 120.0;

// UUT is now ready for adding test steps (covered in next article)
