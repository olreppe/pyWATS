Installing the WATS Client API NuGet Package

The WATS Client API is distributed as NuGet packages, with different packages for different .NET versions. Choose the package that matches your project's target framework.

NuGet Packages

For .NET Core, .NET 6, .NET 8, and newer:

Install-Package Virinco.WATS.ClientAPI -Version 7.*

For .NET Framework 4.7.2 and 4.8:

Install-Package WATS.Client -Version 6.*

Important: The package name is different for .NET Framework vs. .NET Core/8+:

  • .NET Core/8+: Virinco.WATS.ClientAPI
  • .NET Framework 4.8: WATS.Client

Initializing the WATS Client API

Before you can create and submit UUT reports, you must initialize the WATS Client API. The API provides the TDM class (Test Data Model) which is your main interface for all operations.

Basic Initialization

Create and initialize the API client:

using Virinco.WATS.Interface;

// Create the API instance
TDM api = new TDM();

// Initialize in mock mode (for testing without server)
api.InitializeAPI(true);  // true = mock mode

// Initialize in production mode
api.InitializeAPI(false); // false = production mode

Validation Mode

Set the validation mode to control how the API handles data that exceeds database limits: 

// AutoTruncate: Automatically truncate strings that exceed database limits
api.ValidationMode = ValidationModeType.AutoTruncate;

// Strict: Throw exception if data exceeds limits
api.ValidationMode = ValidationModeType.Strict;

Recommendation: Use AutoTruncate to prevent submission failures due to unexpectedly long strings in your test data.

Namespace Reference

Add the following using statement to your code:

using Virinco.WATS.Interface;

This namespace contains all the classes and enumerations you need for creating UUT reports, including:

  • TDM - The main API client
  • UUTReport - The UUT report object
  • UUTStatusType - UUT status enumeration (Passed, Failed, etc.)
  • StepStatusType - Step status enumeration
  • ValidationModeType - Validation mode enumeration
