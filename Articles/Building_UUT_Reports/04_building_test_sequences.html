Building Test Sequences

After creating the UUT report header, you need to build the test sequence by adding test steps. Every UUT report has a root sequence that contains all test steps.

Getting the Root Sequence

The root sequence is the top-level container for all test steps:

// Get the root sequence for the UUT
SequenceCall rootSequence = uut.GetRootSequenceCall();

Adding a Pass/Fail Step

A Pass/Fail step represents a simple boolean test result (passed or failed) with no numeric measurement. 

// Add a Pass/Fail step to the root sequence
PassFailStep continuityStep = rootSequence.AddPassFailStep("Continuity Test");

// Add the test result (true = passed, false = failed)
continuityStep.AddTest(
    measurement: true,
    status: StepStatusType.Passed
);

// Optional: Add a comment
continuityStep.ReportText = "All connections verified";

Adding a String Value Step

A String Value step stores text data, such as firmware versions, serial numbers, or configuration values. 

// Add a String Value step
StringValueStep firmwareStep = rootSequence.AddStringValueStep("Firmware Version");

// Add the string value
firmwareStep.AddTest(
    measurement: "v2.1.5",
    status: StepStatusType.Passed
);

// You can also use it for verification
StringValueStep serialStep = rootSequence.AddStringValueStep("Read Serial Number");
serialStep.AddTest(
    measurement: "SN123456",
    status: StepStatusType.Passed // or Failed if verification failed
);

Adding Multiple Steps in Sequence

Build your test sequence by adding steps in the order they were executed:

// Get root sequence
SequenceCall rootSeq = uut.GetRootSequenceCall();

// Step 1: Power On
PassFailStep powerOn = rootSeq.AddPassFailStep("Power On");
powerOn.AddTest(true, StepStatusType.Passed);

// Step 2: Firmware Version Check
StringValueStep fwVersion = rootSeq.AddStringValueStep("Firmware Version");
fwVersion.AddTest("v2.1.5", StepStatusType.Passed);

// Step 3: Continuity Tests
PassFailStep continuity = rootSeq.AddPassFailStep("Continuity Test");
continuity.AddTest(true, StepStatusType.Passed);

// Step 4: Communication Test
PassFailStep comm = rootSeq.AddPassFailStep("RS232 Communication");
comm.AddTest(true, StepStatusType.Passed);

Step Status Types

Each step must have a status that indicates the test result:

  • StepStatusType.Passed - Test passed successfully
  • StepStatusType.Failed - Test failed (within limits but failed for other reasons)
  • StepStatusType.Done - Test completed (no pass/fail criteria)
  • StepStatusType.Skipped - Test was skipped
  • StepStatusType.Error - Test encountered an error
  • StepStatusType.Terminated - Test was terminated

Adding Comments and Details

You can add additional information to any step:

PassFailStep step = rootSeq.AddPassFailStep("Communication Test");
step.AddTest(false, StepStatusType.Failed);

// Add a detailed comment
step.ReportText = "No response from device after 3 retry attempts. Check cable connections.";

// Set step start time (optional)
step.StepStartDateTime = DateTime.Now;

Complete Example

using System;
using Virinco.WATS.Interface;

// ... (UUT creation code from previous article)

// Get the root sequence
SequenceCall rootSeq = uut.GetRootSequenceCall();

// Add Pass/Fail step
PassFailStep powerStep = rootSeq.AddPassFailStep("Apply Power");
powerStep.AddTest(true, StepStatusType.Passed);
powerStep.ReportText = "5V applied successfully";

// Add String Value step
StringValueStep versionStep = rootSeq.AddStringValueStep("Read Firmware Version");
versionStep.AddTest("FW-v2.1.5", StepStatusType.Passed);

// Add Pass/Fail step with failure
PassFailStep commStep = rootSeq.AddPassFailStep("UART Communication");
commStep.AddTest(false, StepStatusType.Failed);
commStep.ReportText = "Timeout waiting for ACK";

// Set overall UUT status based on step results
uut.Status = UUTStatusType.Failed;
